# AlgoMind 前后端对接开发文档

## 1. 文档目标

本版本文档用于前后端联调，聚焦以下内容：
- 每个接口请求时前端会抛出什么（请求参数、Header、异常处理）。
- 每个接口后端应返回什么（统一响应结构、字段类型、业务码）。
- 响应数据结构（列表、详情、分页、判题结果、AI分析结果等）。

说明：
- 本项目当前前端包含本地兜底逻辑（localStorage + mock）。
- 为了稳定联调，建议后端优先满足本文档的字段结构；前端即可无痛切换真实接口。

---

## 2. 基础约定

### 2.1 Base URL

前端 Axios 配置位于 `src/api/index.js`：
- `baseURL = import.meta.env.VITE_API_BASE_URL || '/api'`
- 默认请求路径都在 `/api` 下。

### 2.2 超时与认证

- 超时：`10000ms`
- 前端请求拦截器会从 `localStorage.user` 读取 `token`，并自动注入：
  - `Authorization: Bearer <token>`

后端要求：
- 需要鉴权的接口校验 `Authorization`。
- 无 token 或 token 无效时，返回统一错误结构（见 2.4）。

### 2.3 Content-Type

- 默认：`application/json`

### 2.4 统一响应结构（强约束）

前端当前按如下结构判断成功：`res.data.code === 0`。

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失败响应：

```json
{
  "code": 40001,
  "message": "参数错误",
  "data": null
}
```

推荐业务码：
- `0`：成功
- `40001`：参数错误
- `40003`：未登录/Token失效
- `40004`：无权限
- `40400`：资源不存在
- `40900`：资源冲突（如重复提交）
- `50000`：服务内部错误

### 2.5 错误抛出约定（前端会抛出什么）

前端 Axios 响应拦截器对非 2xx 或网络错误执行 `Promise.reject(error)`。

后端注意：
- 若返回 4xx/5xx，前端会进入 `catch`，并触发页面兜底逻辑（本地数据）。
- 若希望前端进入成功分支，必须 HTTP 200 且 `code = 0`。

前端捕获到的常见异常类型：
- 网络异常（断网、跨域、超时）
- HTTP 状态异常（401/403/404/500）
- 业务失败（HTTP 200 但 `code !== 0`）

## 3. 数据模型

### 3.1 User 用户

```json
{
  "id": 1,
  "name": "游客",
  "token": "jwt-token",
  "bio": "先把基础打扎实，再冲更高难度。",
  "gender": "unknown",
  "avatar": "https://...",
  "targetTrack": "algo",
  "weeklyGoal": 10,
  "email": "user@example.com",
  "github": "https://github.com/xxx",
  "website": "https://xxx.com"
}
```

字段说明：
- `targetTrack` 枚举：`algo | ds | contest`
- `weeklyGoal`：number

### 3.2 Level 关卡

```json
{
  "id": 1,
  "track": "algo",
  "order": 1,
  "name": "A1 时间复杂度基础",
  "isUnlocked": true,
  "isCompleted": false,
  "rewardPoints": 18,
  "type": "single",
  "question": "题目内容",
  "options": ["A", "B", "C"],
  "answer": "B",
  "description": "题目描述",
  "stars": 0,
  "bestStars": 0,
  "attemptCount": 0,
  "bestTimeMs": null,
  "targetTimeMs": 90000
}
```

字段说明：
- `type` 枚举：`single | multi | judge | fill | code`
- `answer`：
  - 单选/判断/填空：string
  - 多选：string[]
  - 代码题：关键字数组 string[]（当前前端本地模式这样用）

### 3.3 ForumPost 论坛帖子

```json
{
  "id": 1001,
  "author": "匿名用户",
  "authorLevel": "LV.1",
  "avatar": "",
  "topic": "帖子标题",
  "content": "帖子正文",
  "quote": "引用文本",
  "tag": "# 学习交流",
  "likes": 0,
  "comments": 0,
  "liked": false,
  "createdAt": 1775000000000
}
```

### 3.4 ErrorItem 错题

```json
{
  "id": 2001,
  "levelId": 3,
  "question": "题目标题",
  "userAnswer": "用户答案",
```

## 4. 接口清单（按前端真实调用）

## 4.1 登录

- 方法：`POST`
- 路径：`/api/login`
- 调用位置：`src/stores/user.js`

请求体：

```json
{
  "username": "guest",
  "password": "guest"
}
```

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "user": {
      "id": 1,
      "name": "游客",
      "token": "jwt-token",
      "bio": "先把基础打扎实，再冲更高难度。",
      "gender": "unknown",
      "avatar": "",
      "targetTrack": "algo",
      "weeklyGoal": 10
    },
    "points": 120
  }
}
```

失败响应：

```json
{
  "code": 40001,
  "message": "用户名或密码错误",
  "data": null
}
```

前端抛出/处理：
- `code !== 0` 或请求异常时，登录函数会返回 `false` 或进入本地模拟登录兜底。

## 4.2 获取关卡列表

- 方法：`GET`
- 路径：`/api/levels`
- 调用位置：`src/stores/level.js`

请求参数：无

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "id": 1,
      "track": "algo",
      "order": 1,
      "name": "A1 时间复杂度基础",
      "isUnlocked": true,
      "isCompleted": false,
      "rewardPoints": 18,
      "type": "single",
      "question": "以下哪一个是二分查找在有序数组中的时间复杂度？",
      "options": ["O(n)", "O(log n)", "O(n log n)", "O(1)"],
      "answer": "O(log n)",
      "description": "理解查找算法复杂度",
      "stars": 0,
      "bestStars": 0,
      "attemptCount": 0,
      "bestTimeMs": null,
      "targetTimeMs": 90000
    }
  ]
}
```

失败响应：

```json
{
  "code": 50000,
  "message": "获取关卡失败",
  "data": null
}
```

前端抛出/处理：
- 请求失败时走本地关卡数据。
