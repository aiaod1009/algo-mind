## 算法闯关助手 前后端接口与开发文档（对接版）

本文档用于前后端统一接口设计与响应格式。
目标：后端按本文档实现接口后，前端可直接联调切换，减少字段反复修改。

---

## 1. 项目概览

### 1.1 技术栈

- 前端框架：Vue 3 + Vite
- UI：Element Plus
- 状态管理：Pinia
- HTTP：Axios
- 路由：Vue Router

### 1.2 页面与路由

| 路径 | 页面 | 功能 |
| --- | --- | --- |
| `/` | Login | 登录入口 |
| `/home` | Home | 学习计划、赛道目标 |
| `/courses` | Courses | 网课列表与筛选 |
| `/levels` | Levels | 选关地图 |
| `/challenge/:id` | Challenge | 作答与判题 |
| `/errors` | Errors | 错题本与分析 |
| `/forum` | Forum | 发帖、浏览、点赞、评论计数 |
| `/ranking` | Ranking | 积分排行 |
| `/profile` | Profile | 用户资料编辑 |

说明：除 `/` 外，全部页面要求登录态。

## 2. 全局约定

### 2.1 接口前缀

- 统一前缀：`/api`
- 生产环境地址通过环境变量配置

### 2.2 认证方式

- 登录成功后返回 `token`
- 前端后续请求头：

```http
Authorization: Bearer <token>
```

### 2.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "c9d3a8f7-0fbc-4b8d-a1ea-7f5514d7b8f2",
  "timestamp": "2026-03-28T10:32:30.000Z"
}
```

字段说明：

- `code`: 0 成功，非 0 失败
- `message`: 可展示给用户的提示
- `data`: 业务数据
- `requestId`: 链路追踪 ID，便于排查
- `timestamp`: 服务器时间，ISO 8601 格式

### 2.4 分页结构（列表接口）

```json
{
  "list": [],
  "page": 1,
  "pageSize": 20,
  "total": 120
}
```

### 2.5 时间字段格式

- 推荐统一：ISO 8601（UTC）
- 示例：`2026-03-28T10:32:30.000Z`

## 3. 错误码建议

| code | 含义 | 说明 |
| --- | --- | --- |
| 0 | 成功 | 请求成功 |
| 40001 | 参数错误 | 缺少字段或格式错误 |
| 40002 | 业务校验失败 | 如答案格式不合法 |
| 40101 | 未登录/Token 失效 | 需要重新登录 |
| 40301 | 无权限 | 非本人操作等 |
| 40401 | 资源不存在 | 如关卡或帖子不存在 |
| 40901 | 状态冲突 | 如重复点赞 |
| 50001 | 服务异常 | 后端内部错误 |

## 5. 必做接口（当前前端已接入）

> 下列接口是当前前端直接调用的接口，后端优先实现。

### 5.1 登录

- `POST /api/login`

请求体：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应 `data`：

```json
{
  "user": {
    "id": 2,
    "name": "管理员",
    "token": "admin-token",
    "bio": "持续刷题，稳定提分。",
    "gender": "unknown",
    "avatar": "",
    "targetTrack": "algo",
    "weeklyGoal": 12
  },
  "points": 120
}
```

---

### 5.2 获取关卡列表

- `GET /api/levels`

可选 query：

- `track=algo|ds|contest`（后端可支持过滤，前端可本地分组）

响应 `data`：`Level[]`

---

### 5.3 提交答案

- `POST /api/submit`

请求体：

```json
{
  "levelId": 2,
  "answer": "双指针夹逼"
}
```

说明：

- `answer` 允许字符串或数组
- 多选题建议传数组

响应 `data`：

```json
{
  "correct": true,
  "pointsEarned": 24,
  "nextLevelUnlocked": true
}
```

---

### 5.4 错题列表

- `GET /api/errors`

响应 `data`：`ErrorItem[]`

---

### 5.5 新增错题

- `POST /api/errors`

请求体：

```json
{
  "levelId": 2,
  "question": "在有序数组中寻找两数之和...",
  "userAnswer": "暴力双循环",
  "description": "关卡：A2 双指针技巧。你的答案：暴力双循环",
  "createdAt": "2026-03-28T09:20:00.000Z"
}
```

响应 `data`：

```json
{
  "id": 10001
}
```

---

### 5.6 错题 AI 分析

- `POST /api/error-analysis`

请求体：

```json
{
  "errorId": 10001,
  "description": "关卡：A2 双指针技巧。你的答案：暴力双循环"
}
```

响应 `data`：

```json
{
  "analysis": "错误原因可能在于对题目条件理解不完整..."
}
```

---

### 5.7 论坛帖子列表

- `GET /api/forum-posts`

建议 query：

- `page`、`pageSize`
- `sort=latest|hot`

响应 `data`：可返回分页结构，`list` 内为 `ForumPost[]`

---

### 5.8 发布帖子

- `POST /api/forum-posts`

请求体：

```json
{
  "author": "管理员",
  "authorLevel": "LV.3",
  "avatar": "",
  "topic": "校招迷茫期如何破局",
  "content": "最近投递节奏有点乱，分享下我的调整方法...",
  "quote": "学长：先稳住节奏，再冲结果",
  "tag": "# 学习交流"
}
```

响应 `data`：返回创建后的 `ForumPost`

## 6. 建议扩展接口（便于后续去本地兜底）

> 下列接口不是当前前端强依赖，但建议后端同时设计，减少后续改动。

### 6.1 获取课程列表

- `GET /api/courses?track=algo|ds|contest&page=1&pageSize=20`

响应 `data`：分页结构，`list` 为 `Course[]`

### 6.2 点赞帖子

- `POST /api/forum-posts/{id}/like`

请求体：

```json
{
  "action": "like"
}
```

响应 `data`：

```json
{
  "likes": 521,
  "liked": true
}
```

### 6.3 评论计数（简版）

- `POST /api/forum-posts/{id}/comment-count`

响应 `data`：

```json
{
  "comments": 30
}
```

### 6.4 用户资料更新

- `PUT /api/users/me`

请求体：

```json
{
  "name": "管理员",
  "bio": "持续刷题，稳定提分。",
  "gender": "unknown",
  "avatar": "",
  "targetTrack": "algo",
  "weeklyGoal": 12
}
```

### 6.5 排行榜

- `GET /api/ranking?track=all|algo|ds|contest&period=all|week|month`

响应 `data`：

```json
[
  {
    "id": 2,
    "name": "管理员",
    "points": 120,
    "targetTrack": "algo",
    "rank": 1
  }
]
```

## 7. 前端联调要求（后端需关注）

1. 字段命名请保持稳定，不要中英文混用。
2. 所有列表接口返回空数组，不返回 `null`。
3. `code != 0` 时请返回明确 `message`，前端直接透出。
4. 建议所有写操作返回最新对象，避免前端二次拼装。
5. 跨域请在测试环境允许前端域名访问。

## 8. 环境与部署

- 前端环境变量：

```env
VITE_API_BASE_URL=http://localhost:3000/api
```

- 本地启动：

```bash
npm install
npm run dev
```

- 生产构建：

```bash
npm run build
```

## 9. 对接清单（给后端）

后端首批实现顺序建议：

1. 登录：`POST /login`
2. 关卡：`GET /levels`、`POST /submit`
3. 错题：`GET /errors`、`POST /errors`、`POST /error-analysis`
4. 论坛：`GET /forum-posts`、`POST /forum-posts`
5. 扩展：课程/点赞/评论/用户资料/排行榜

当以上接口按本文档返回后，前端即可去除大部分本地兜底逻辑并切到真实数据流。
