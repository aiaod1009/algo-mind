# AlgoMind 开发文档

## 项目概述

**AlgoMind** 是一个面向算法学习的在线教育平台，采用 Vue 3 + Vite 技术栈构建。平台提供关卡挑战、视频课程、社区论坛、排行榜、错题本等功能，采用游戏化的学习方式帮助用户提升算法能力。

---

## 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | Vue 3 (Composition API) |
| 构建工具 | Vite 6.2.0 |
| 状态管理 | Pinia 3.0.1 |
| 路由 | Vue Router 4.5.0 |
| HTTP 客户端 | Axios 1.8.4 |
| 代码规范 | ESLint 9.21.0 |

## 核心模块详解

### 1. 路由系统 (src/router/index.js)

路由采用 Vue Router 4 实现，支持路由守卫进行登录验证。

**路由表：**

| 路径 | 组件 | 说明 | 登录要求 |
|------|------|------|----------|
| /login | Login | 登录页面 | 否 |
| / | Home | 首页/仪表盘 | 是 |
| /levels | Levels | 关卡选择 | 是 |
| /challenge/:id | Challenge | 代码挑战 | 是 |
| /courses | Courses | 视频课程 | 是 |
| /forum | Forum | 学习社区 | 是 |
| /ranking | Ranking | 排行榜 | 是 |
| /errors | Errors | 错题本 | 是 |
| /profile | Profile | 个人中心 | 是 |

**路由守卫逻辑：**
- 检查 `localStorage` 中的 `isLoggedIn` 状态
- 未登录用户访问受保护路由时重定向到登录页
- 已登录用户访问登录页时重定向到首页

### 2. 状态管理 (Pinia Stores)

#### 2.1 用户状态 (src/stores/user.js)

管理用户登录状态、积分和用户信息。

**State：**
```javascript
{
  isLoggedIn: boolean,      // 登录状态
  points: number,           // 用户积分
  userInfo: Object | null   // 用户信息（名称、邮箱、等级等）
}
```

**核心方法：**
- `login(userData)` - 用户登录，持久化到 localStorage
- `logout()` - 用户登出，清除所有状态
- `setPoints(value)` - 设置积分
- `addPoints(value)` - 增加积分
- `deductPoints(value)` - 扣除积分
- `updateUserInfo(info)` - 更新用户信息

#### 2.2 关卡状态 (src/stores/level.js)

管理关卡进度、当前关卡和关卡数据。

**State：**
```javascript
{
  currentLevel: number,     // 当前关卡 ID
  completedLevels: number[], // 已完成关卡列表
  levelStatus: Object       // 各关卡状态（locked/unlocked/completed）
}
```

**核心方法：**
- `setCurrentLevel(id)` - 设置当前关卡
- `completeLevel(id)` - 标记关卡完成
- `unlockLevel(id)` - 解锁关卡
- `isLevelCompleted(id)` - 检查关卡是否完成
- `isLevelUnlocked(id)` - 检查关卡是否解锁
- `getLevelStatus(id)` - 获取关卡状态
- `resetProgress()` - 重置进度

#### 2.3 错题本状态 (src/stores/error.js)

管理用户错题记录。

**State：**
```javascript
{
  errorQuestions: Array     // 错题列表
}
```

**核心方法：**
- `addError(question)` - 添加错题
- `removeError(id)` - 移除错题
- `clearErrors()` - 清空错题本
- `hasError(id)` - 检查是否已存在
- `getErrorStats()` - 获取错题统计

#### 2.4 论坛状态 (src/stores/forum.js)

管理论坛帖子、评论和互动。

**State：**
```javascript
{
  posts: Array,             // 帖子列表
  currentPost: Object|null, // 当前查看的帖子
  categories: Array,        // 分类列表
  filters: Object           // 筛选条件
}
```

**核心方法：**
- `addPost(post)` - 发布帖子
- `deletePost(id)` - 删除帖子
- `addComment(postId, comment)` - 添加评论
- `likePost(id)` - 点赞帖子
- `setCategory(category)` - 设置分类筛选
- `setSort(sort)` - 设置排序方式
- `searchPosts(keyword)` - 搜索帖子

### 3. API 层 (src/api/index.js)

封装所有后端接口调用，目前使用模拟数据。

**API 列表：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/login | 用户登录 |
| POST | /api/auth/register | 用户注册 |
| POST | /api/auth/logout | 用户登出 |
| GET | /api/user/profile | 获取用户信息 |
| PUT | /api/user/profile | 更新用户信息 |
| GET | /api/levels | 获取关卡列表 |
| GET | /api/levels/:id | 获取关卡详情 |
| POST | /api/levels/:id/submit | 提交代码 |
| GET | /api/courses | 获取课程列表 |
| GET | /api/courses/:id | 获取课程详情 |
| GET | /api/forum/posts | 获取帖子列表 |
| POST | /api/forum/posts | 发布帖子 |
| GET | /api/ranking | 获取排行榜 |
| GET | /api/errors | 获取错题本 |

### 4. 全局常量 (src/constants.js)

定义关卡数据结构，每个关卡包含：

```javascript
{
  id: number,               // 关卡 ID
  title: string,            // 标题
  description: string,      // 描述
  difficulty: string,       // 难度（入门/进阶/高级）
  tags: string[],           // 标签
  initialCode: string,      // 初始代码模板
  testCases: Object[],      // 测试用例
  hints: string[],          // 提示
  solution: string          // 参考答案
}
```

## 开发规范

### 代码风格

- 使用 Composition API (`<script setup>`)
- 组件名使用 PascalCase
- 文件名使用 PascalCase（组件）或 camelCase（JS）
- 使用 2 空格缩进
- 使用单引号

### 状态管理规范

- 所有状态变更通过 Store 方法
- 持久化状态使用 `localStorage`
- 状态变更后触发 UI 更新

### 组件开发规范

- Props 使用 `defineProps` 定义类型
- Emits 使用 `defineEmits` 定义
- 使用 `ref` 和 `computed` 管理响应式数据
- 生命周期钩子使用 `onMounted`、`onUnmounted`

## 数据流说明

```
用户操作 → 组件事件 → Store 方法 → State 更新 → UI 响应
                ↓
            API 调用（模拟/真实）
                ↓
            localStorage 持久化
```

---

## 注意事项

1. **登录状态**：刷新页面后登录状态通过 localStorage 恢复
2. **数据持久化**：用户进度、错题本等数据存储在 localStorage
3. **响应式设计**：页面适配移动端（最大宽度限制）
4. **模拟数据**：当前使用模拟数据，后续需对接真实 API
